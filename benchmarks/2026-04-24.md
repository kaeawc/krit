## Findings note

Warm runs (7,550) vs cold runs (7,592) differ by 42 findings. The warm KAA run loads
types.json produced by the previous cached run, which may have a slightly different
oracle filter fingerprint than the cold run's full JVM invocation. Not a regression —
both are correct for their respective oracle inputs.

## Benchmark command

```bash
# Cold KAA (matches this run):
rm -f /path/to/project/.krit/types.json
./krit -no-cache -no-parse-cache -no-cache-oracle -perf -f json -q /path/to/project/

# Or use the script (2 cold runs):
scripts/benchmark-oracle.sh /path/to/project 2
```

Note: `benchmark-oracle.sh` deletes `.krit/types.json` but does not delete `types-cache/`.
The `-no-cache-oracle` flag bypasses the per-file store regardless, so this has no effect
on cold-run accuracy.
